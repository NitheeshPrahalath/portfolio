# DSA PROGRESSION PLAN

## Core Rule (MOST IMPORTANT)
Do NOT binge 5 random problems. Instead:

1. **Understand brute force** — why does it work?
2. **Optimize logically** — think out loud before coding
3. **Code without looking at solution**
4. **Explain aloud** — as if to an interviewer
5. **Re-solve after 3 days, then after 7 days** — this is how memory forms

> 1–2 deeply understood problems > 10 passively copied solutions.

# INTERVIEW STRATEGY

## The Golden Answer Structure (Use this for EVERY technical answer)

> **Problem → Architecture → Decision → Tradeoff → Result**

**BAD:** "We used threading."

**GOOD:** "We separated camera ingestion, AI inference, and event generation into independent threads to prevent blocking latency during real-time processing. This ensured inference didn't wait on I/O."

That second answer sounds senior. It sounds like someone who made real engineering decisions.

---

## Car Interceptor — Perfect Explanation Script

Use this structure in interviews:

**Step 1 — Problem:**
"We needed a mobile edge-based traffic violation detection system running in real time from cameras mounted on vehicles."

**Step 2 — Input Pipeline:**
"We processed RTSP/OpenCV frames from 3 simultaneous camera streams on NVIDIA Jetson edge devices."

**Step 3 — Inference:**
"Frames were passed into YOLOv7 inference pipelines accelerated with TensorRT on the Jetson GPU."

**Step 4 — Violation Logic:**
"Post-inference metadata was combined with 4D radar speed data and GPS coordinates to identify violations — speeding, no helmet, triple riding, wrong entry, no parking."

**Step 5 — Event System:**
"Violation events were generated with plate number, speed, timestamp, GPS coordinates, captured GIF evidence, and cloud sync status — uploaded to AWS S3."

**Step 6 — YOUR Contribution (say this clearly):**
"I personally built the backend feature additions, data synchronization pipeline, triple-camera support, GPS geo-zone no-parking logic, data retention, and all frontend functionality. I also handled production bug fixes across the codebase."

# AI USAGE GUIDE (Important Mindset Shift)

**BAD way to use AI:**
"Build me the entire video downloader app."

**CORRECT way to use AI:**
- "Explain why this useEffect is causing infinite re-render."
- "What's a better folder structure for a FastAPI project with queues?"
- "Review my threading architecture — what could go wrong?"
- "Why is my yt-dlp subprocess not releasing after download?"
- "Explain hydration error in Next.js."

**Use AI as a senior engineer mentor. Not a code generator.**

The difference: when you use AI correctly, you understand the output. When you use it as a code generator, you can't explain it in interviews — and interviewers will expose this.
